# Document Template Policy

Document templates should help users organize drafts without pretending to be final legal submissions.

## Template categories

- timeline
- material list
- public petition
- demand letter
- civil complaint outline
- internal review memo
- business compliance memo
- research source note

## Required metadata

Every template should include:

- draft status
- jurisdiction
- user mode
- source verification status
- generated date
- review warning

## Required sections by template

### Timeline

- date
- event
- linked material
- verification status

### Material list

- material id
- title
- source
- date
- supports which fact
- note

### Public petition

- recipient
- summary
- timeline
- requested action
- source candidates
- attached materials
- review checklist

### Demand letter

- recipient
- sender placeholder
- factual summary
- requested action
- response deadline placeholder
- material list
- source candidates
- reservation note

### Civil complaint outline

- parties
- requested outcome
- factual background
- basis candidates
- harm summary
- link between facts and request
- material list
- source candidates
- attachment list
- review checklist

### Internal review memo

- issue summary
- confirmed facts
- unconfirmed facts
- source candidates
- risk notes
- recommended next check
- recordkeeping note

## Template safety rule

Templates should use placeholders for private data. They should not include real private values in repository examples.

## Export rule

Markdown is the first export format. DOCX and PDF can be added later after template quality is stable.
