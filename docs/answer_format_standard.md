# Standard Answer Format

This project should answer legal-source questions in a predictable structure for citizens, public officials, business users, and researchers.

## Core answer structure

Every legal-source answer should follow this order:

1. Question understood
2. Scope and limits
3. Source status
4. Short answer
5. Relevant rule candidates
6. Application notes
7. Risk and uncertainty
8. Next action checklist
9. Citation metadata

## 1. Question understood

Restate the user's question in plain language.

Example:

```text
You are asking whether a service interruption can support a compensation request and which official sources should be checked first.
```

## 2. Scope and limits

State what the answer can and cannot do.

```text
This is a source-based preparation answer. It is not final legal advice or representation.
```

## 3. Source status

Classify the source basis:

- verified official source
- source candidate only
- user-provided material
- unverified claim
- missing source

## 4. Short answer

Give a concise answer without overclaiming.

## 5. Relevant rule candidates

List statutes, articles, cases, decisions, or agency materials as candidates.

Each item should include:

- title
- identifier
- source authority
- date or effective date
- retrieved date
- reliability grade
- limitation note

## 6. Application notes

Apply the source candidates to the user's situation carefully.

Use language such as:

- may be relevant
- should be verified
- depends on evidence
- requires official source confirmation

Avoid:

- guaranteed
- certainly wins
- final conclusion

## 7. Risk and uncertainty

List weak points:

- missing dates
- missing material
- unclear source
- possible opposing argument
- jurisdiction uncertainty

## 8. Next action checklist

Give concrete next steps:

- collect notice
- collect contract
- save screenshots
- request official record
- verify article text
- prepare draft

## 9. Citation metadata

Every formal citation candidate should include metadata. If metadata is missing, mark it clearly.

## User-type variants

### Citizen mode

Focus on plain language, timeline, materials, and next steps.

### Public official mode

Focus on neutral facts, authority, source reliability, decision memo structure, and administrative record.

### Business mode

Focus on compliance scope, operational risk, source candidates, internal checklist, and documentation.

### Research mode

Focus on source classification, comparison, historical version, and data reliability.

## Rule

The same source can produce different answer styles, but the source metadata and safety boundary must remain the same.
