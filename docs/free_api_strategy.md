# Free API Strategy

The project should remain useful with free public resources and local tools.

## Goals

- reduce cost for ordinary users
- avoid vendor lock-in
- keep sensitive case material local by default
- use official public data whenever possible
- allow optional external AI APIs for drafting support

## Source priority

1. official public legal database
2. public agency materials
3. trusted public institution materials
4. secondary sources for background only
5. user-provided material needing verification

## External AI use

External AI APIs may help with:

- summarization
- structure extraction
- plain-language rewriting
- draft polishing
- counterargument brainstorming

They should not be the source of legal truth.

## Local-first default

A user should be able to:

- store case notes locally
- create timelines locally
- build item lists locally
- use official source adapters locally
- export draft documents locally

## Cost rule

Free API access is helpful, but the system should not depend on a single free tier staying free forever.

Any paid or external dependency should be optional.
