# API Strategy

The project should remain useful even without paid services.

## Default mode

- Store case data locally.
- Use local files and local database first.
- Use official public legal information sources where available.
- Treat external LLM APIs as optional helpers, not required infrastructure.

## Korean source strategy

For Korea, the first source adapter targets public legal information services such as:

- National Law Information Center: https://www.law.go.kr
- Open Law API portal: https://open.law.go.kr

The adapter should keep the original response where possible and create a normalized source envelope.

## External LLM strategy

External LLM APIs can help with summarization and drafting. They should be used only after:

1. masking private details,
2. previewing outgoing text,
3. user approval,
4. recording whether the result was generated by an external model.

## Cache policy

Official source responses may be cached locally with:

- source URL
- retrieved date
- effective date if available
- hash of source text
- normalized citation label

Case evidence should not be stored in the repository.
