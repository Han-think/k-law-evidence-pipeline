# Drafting Boundary

This project can help draft documents, but it must keep a clear boundary.

## Allowed assistance

- organize user notes
- build a timeline
- list materials
- suggest issue categories
- retrieve official source candidates
- check citation metadata
- create draft text
- list missing materials
- list possible counterarguments

## Not allowed as a system claim

- guaranteed result
- final legal opinion
- final claim amount
- automatic filing decision
- professional representation

## Draft labels

Every generated document should clearly indicate one of these statuses:

- draft
- needs source verification
- needs human review
- ready for professional review

## Recommended document flow

```text
case summary
→ timeline
→ material list
→ issue candidates
→ source candidates
→ draft document
→ review checklist
```

## Tone rule

Drafts should be direct, factual, and structured. Emotional language may be preserved in a separate note, but official drafts should separate fact, claim, and requested outcome.
